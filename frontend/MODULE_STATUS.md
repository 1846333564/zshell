# 前端模块状态

## 范围

Wails 桌面窗口中的 Vue/Vite 前端。

## 重要文件

- `src/App.vue`
- `src/style.css`
- `src/components`
- `src/services`

## 当前状态

UI 是双栏桌面布局，包含自定义无边框窗口顶栏、左侧监控面板，以及右侧连接标签、终端和文件区域。未连接首页会隐藏监控侧栏，只保留已保存连接和连接配置区域；连接失败信息会在配置面板内换行，不再撑开首页布局。保存连接管理使用后端配置 API，不把 SSH 密码保存在浏览器本地存储，并支持前端、后端、运维三种工作模式。应用壳会屏蔽非文件区系统右键菜单，同时允许文件管理器自定义菜单；非终端 UI 缩放和终端字体大小都通过后端偏好持久化；终端区域会抵消全局 UI zoom，避免 xterm.js 鼠标选择坐标相对光标左上偏移；文件和终端右键菜单渲染在视口层，避免缩放导致菜单坐标偏移；全局滚动条使用应用风格。Monaco Editor 会在应用启动后空闲预热，并在编辑器窗口打开时按需加载，不进入首屏静态 bundle。

`src/App.vue` 现在提供“关于 zShell”弹窗和“检查更新”流程；应用启动后会后台静默检查更新，只有发现新版本时才弹窗；更新弹窗在发现新版本或遇到更新错误时会保留 GitHub Release 下载页入口，确认更新后显示阶段进度、下载字节进度、重试和校验日志。文件区域包含 Monaco 在线文本编辑，编辑器以多个普通浮动窗口显示，支持代码高亮、Tab 缩进、`Ctrl+F` 搜索、替换、拖动、最小化、最大化、保存和脏关闭处理；连接成功后会使用后端返回的服务器硬件信息。文件管理器按连接工作模式默认进入 `/var`、`/opt` 或 `/`，访问过的目录会先用内存缓存渲染，再刷新真实远端目录，并提供路径历史、全局已知目录 Tab 唯一补全、按 CPU 线程数并发的一层目录预加载、可折叠上传记录和真实远端上传进度。

## 已知工作

较大布局修改后，需要在打包后的 Wails 窗口中做视觉冒烟检查。更新弹窗需要配合真实 GitHub Release 验证完整下载和重启流程。
