# 前端模块状态

## 范围

Wails 桌面窗口中的 Vue/Vite 前端。

## 重要文件

- `src/App.vue`：应用模板壳。
- `src/composables/app`：连接标签、更新弹窗、UI 缩放和窗口控制组合函数。
- `src/style.css` 与 `src/styles`：全局样式入口和分包样式。
- `src/components`
- `src/services`

## 当前状态

UI 是双栏桌面布局，包含自定义无边框窗口顶栏、左侧监控面板，以及右侧连接标签、终端和文件区域。未连接首页会隐藏监控侧栏，只保留已保存连接和连接配置区域；连接失败信息会在配置面板内换行，不再撑开首页布局。保存连接管理使用后端配置 API，不把 SSH 密码保存在浏览器本地存储，并支持前端、后端、运维三种工作模式；应用启动后短暂等待就绪并在配置加载或连接进行中禁用连接动作，避免刚打开窗口时快速点击造成重复连接或渲染高负载。应用壳会屏蔽非文件区系统右键菜单，同时允许文件管理器自定义菜单；非终端 UI 缩放、终端字体大小、主题键、自定义主题颜色和 GPU 渲染开关都通过后端偏好持久化；`UI管理` 可直接切换 GPU 渲染并提示重启生效，默认保持开启。默认主题保持原有 wiShell 配色，`UI管理 -> 主题设置` 可切换 Dracula、Nord、Tokyo Night、Catppuccin Mocha、Gruvbox Dark、One Dark、Solarized Dark 或自定义颜色，并同步影响应用外壳、xterm 终端和 Monaco 编辑器；终端区域会抵消全局 UI zoom，避免 xterm.js 鼠标选择内容时选区相对鼠标位置左上偏移，并保持正常 100% flex 尺寸，避免缩放二次参与布局造成右侧或下方空白；文件和终端右键菜单渲染在视口层，避免缩放导致菜单坐标偏移；全局滚动条使用应用风格。Monaco Editor 会在应用启动后延后到更空闲的阶段预热，在线编辑器窗口打开后按需加载，不进入首屏静态 bundle；远程下载期间由只读文本预览增量显示已到达内容，完成后再切换到 Monaco 和文件语言。

`src/App.vue` 现在提供“关于 wiShell”弹窗和“检查更新”流程；应用启动后会后台静默检查更新，只有发现新版本时才弹窗；更新弹窗在发现新版本或遇到更新错误时会保留 GitHub Release 下载页入口，确认更新后显示阶段进度、下载字节进度、重试和校验日志，并在下载、校验等可取消阶段提供“停止”按钮。左侧监控面板 1 秒自动刷新，显示保存的硬件信息、低成本实时系统信息、服务器时间和上下行网速折线图，网速图只显示峰值并带左侧纵轴刻度；刷新期间保留服务器时间且在其后显示请求延迟，不再闪烁“更新中”。文件区域包含 Monaco 在线文本编辑，编辑器以多个普通浮动窗口显示，支持代码高亮、Tab 缩进、`Ctrl+F` 搜索、替换、拖动、最小化、最大化、保存和脏关闭处理；双击或右键编辑会立即创建窗口并持续显示 XHR 原生下载进度和只读增量预览，完成后再把全文交给 Monaco、切换文件语言并开放编辑；关闭未完成窗口会取消浏览器与服务器 SFTP 读取。连接成功后会使用后端返回的服务器硬件信息。文件管理器按连接工作模式默认进入 `/var`、`/opt` 或 `/`，访问过的目录会先用内存缓存渲染，再刷新真实远端目录；左侧目录树仅展示目录且每个目录独占一行，90 度空心折线图标固定在左侧，整行点击会选中并切换展开/折叠，刷新后仍保留折叠状态，右侧文件列表和左树使用固定行高虚拟滚动。目录、文件和空白区域使用独立右键目标及菜单，支持原位重命名、整目录动作、上传、复制/剪切、条件粘贴和复制路径；右键菜单移除顶部路径标题，并按真实渲染高度适配视口。动作完成后会定向刷新目标及同连接源父目录。文件管理器还提供按频次/最近访问排序且默认滚到底部的路径历史、全局已知目录 Tab 唯一补全、当前目录加载完成后立即启动的递归分批目录预加载、手动刷新重索引所有已打开目录、固定面包屑/表头、复用空心折线图标的列头排序动画、可折叠上传记录和真实远端上传进度。

在线编辑不再消费本地 NDJSON/Base64 分块流。`apiClient.js` 使用 XHR 网络进度事件更新真实字节数，并按已交付字符位置从累计 `responseText` 提取新增文本；`useFileManager.js` 从创建起使用 Vue 响应式编辑窗口，在非响应式 pending 缓冲区收集网络块，最多每 1 秒向 raw `contentChunks` 发布一个合并批次并递增 `appendVersion`。下载期间 `RemoteCodeEditor.vue` 把每批内容作为新文本节点追加到可见只读预览，不再反复重建 textarea 全文，Monaco 保持隐藏并在后台初始化；完整响应结束时才统一设置 `content` / `originalContent`、布局 Monaco 并切换显示。每个窗口仍以编辑窗口 ID 管理非响应式 `AbortController`，真正关闭窗口或组件卸载时取消请求，取消不会显示错误或重新发起普通读取。

前端大文件已按职责拆分：`App.vue` 的业务状态移入 `src/composables/app`，`FileManager.vue` 的 setup 移入 `src/components/useFileManager.js`，全局样式拆入 `src/styles`，文件管理路径/格式化/常量工具拆入 `src/components/fileManagerUtils.js`。`useFileManager.js` 当前已恢复为可读多行源码，方便继续拆分目录树、预加载、编辑器窗口和文件动作逻辑；下一步应把这些职责拆到更小组合函数中。
