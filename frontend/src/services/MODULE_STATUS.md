# 前端服务模块状态

## 范围

HTTP API、下载、上传、监控快照、跨服务器传输、更新接口和终端 WebSocket 的前端封装。

## 重要文件

- `apiClient.js`
- `wsClient.js`
- `editorWarmup.js`

## 当前状态

API 和 WebSocket 调用在 Wails 中使用注入的后端基础地址，在 Vite 开发时使用相对路径。`apiClient.js` 包含保存连接配置的列表/创建/更新/删除封装，连接 payload 会透传工作模式，连接测试会返回服务器硬件信息；还包含 UI 缩放、终端字体大小、主题键和自定义主题颜色偏好封装、应用信息封装、检查更新和带 NDJSON 流式进度回调的应用更新封装，应用更新请求支持 `AbortController` 停止并能识别后端 stopped 事件；还包含带后端真实 SFTP 写入进度回调的上传封装、带 NDJSON 真实下载进度和 32 KiB 文本 chunk 回调的远程文本读取封装，chunk 数据按 base64 字节流用 `TextDecoder` 流式解码，避免 UTF-8 字符被分块切开时乱码；流式读取如果是 WebView/fetch stream 层失败会回退到普通读取，后端明确返回的远程读取错误仍会向用户显示；还包含可取消的目录列表请求、归档下载、直接下载、远程文本读写，以及文件右键菜单使用的传输和删除封装。`editorWarmup.js` 延后到启动后一段时间并等待空闲回调再预热 Monaco，避免刚打开应用快速连接时与终端、监控和文件管理首轮渲染叠加。

## 已知工作

保存连接凭据不能放回浏览器存储；浏览器本地存储只用于非敏感的临时 UI 状态。
