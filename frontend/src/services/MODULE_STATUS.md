# 前端服务模块状态

## 范围

HTTP API、下载、上传、监控快照、跨服务器传输、更新接口和终端 WebSocket 的前端封装。

## 重要文件

- `apiClient.js`
- `wsClient.js`
- `editorWarmup.js`

## 当前状态

API 和 WebSocket 调用在 Wails 中使用注入的后端基础地址，在 Vite 开发时使用相对路径。`apiClient.js` 包含保存连接配置的列表/创建/更新/删除封装，连接 payload 会透传工作模式，连接测试会返回服务器硬件信息；还包含 UI 缩放、终端字体大小、主题键、自定义主题颜色和 GPU 渲染开关偏好封装、应用信息封装、检查更新和带 NDJSON 流式进度回调的应用更新封装，应用更新请求支持 `AbortController` 停止并能识别后端 stopped 事件；还包含带后端真实 SFTP 写入进度回调的上传封装，以及基于 XHR 原始 UTF-8 响应的远程文本读取封装。XHR 使用原生网络进度事件和 `X-WiShell-File-Size` 预期大小响应头，不在 WebView 主线程解析 NDJSON/Base64，并在完成时拒绝实际字节数不一致的响应；文件路径、名称和修改时间通过 CORS 暴露的 Base64URL 响应头返回，`AbortSignal` 会直接中止请求且不会触发普通读取 fallback。服务还包含可取消的目录列表请求、归档下载、直接下载、远程文本读写、安全原位重命名，以及文件右键菜单使用的复制/移动和删除封装。`editorWarmup.js` 延后到启动后一段时间并等待空闲回调再预热 Monaco，避免刚打开应用快速连接时与终端、监控和文件管理首轮渲染叠加。

远程文本读取在每次 XHR progress 中先按已交付字符位置切出新增 `responseText`，通过 `onChunk` 回调增量文本，同时独立上报真实字节进度；onload 会补发尾段、核对预期字节数并返回完整文本。读取错误按远程错误返回，取消使用 `AbortError`，两者都不会自动开始第二次下载。
