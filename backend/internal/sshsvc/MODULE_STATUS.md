# SSH 服务模块状态

## 范围

SSH 客户端创建、共享连接缓存、认证方式选择、一次性命令执行和交互式 PTY shell。

## 重要文件

- `client.go`
- `shell.go`

## 当前状态

支持密码认证和当前 Windows 用户的 `~/.ssh/id_rsa` 私钥认证。连接测试成功后可读取服务器硬件信息，当前包含 CPU 硬件线程数、核心数、CPU 型号、内存大小和读取时间。文件快路径和监控快照可复用按连接信息和凭据哈希区分的共享 SSH 客户端，减少复制、移动、删除和 1 秒监控等短操作的重复握手成本；共享 SSH 客户端活性探测有短超时，失效连接会被关闭并从缓存移除，避免后续 SFTP 操作卡住等待旧连接。一次性 `ExecCommand` 仍保持独立连接语义。PTY shell 为终端 WebSocket 提供交互式能力，并定期发送 SSH 全局 keepalive 请求，降低长时间交互终端断线概率。

## 已知工作

Host key 校验目前仍是宽松模式。私钥模式有意限制为 `~/.ssh/id_rsa`。如果远程 shell 自己通过 `TMOUT` 等策略强制空闲登出，传输层 keepalive 不会覆盖服务器策略。
