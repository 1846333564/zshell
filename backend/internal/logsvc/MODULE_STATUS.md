# 日志服务模块状态

## 范围

应用日志初始化、按小时轮转、启动清理和错误/panic 记录工具。

## 重要文件

- `logger.go`：日志目录、24 小时清理、小时文件轮转和标准库 `log` 输出接管。
- `events.go`：错误、panic、调用位置和 recover 工具。
- `logger_test.go`：清理和小时轮转单元测试。

## 当前状态

日志目录为当前用户配置目录下的 `%AppData%\zShell\log`，文件名为 `zshell-YYYYMMDD-HH.log`。每次应用启动时会删除修改时间早于 24 小时的 zShell 日志文件；运行中写日志时会按当前小时自动切换文件。日志事件会保留错误位置和原始错误原因，panic 会附带 stack trace。

## 已知工作

日志目前只做本地文件落盘，不提供 UI 查看器或压缩归档。
