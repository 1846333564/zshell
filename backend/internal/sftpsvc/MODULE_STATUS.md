# SFTP 服务模块状态

## 范围

远程目录列表、上传、下载、文本文件读写、zip 归档生成、同服务器快速文件动作和服务器间复制/移动。

## 重要文件

- `service.go`、`upload.go`、`upload_batch.go`、`upload_dirs.go`、`upload_workers.go`、`download.go`、`text.go`、`text_stream.go`、`text_raw.go`、`archive.go`、`transfer.go`、`delete.go`：按 SFTP 功能拆分的服务实现。
- `paths.go`、`types.go`：远程路径工具和共享 DTO。
- `service_test.go`

## 当前状态

支持目录列表、多文件/多目录上传、直接下载、面向在线编辑的 256 MiB 远程文本读写、递归目录归档下载、同父目录安全重命名、跨服务器 SFTP 流式复制/移动，以及同服务器远端 `cp`/`mv` 快速复制/移动和远端 `rm -rf` 快速删除。重命名只接受单个 basename，拒绝空名称、`.`、`..`、斜杠、NUL、根路径、home 简写和已有目标，并使用不带覆盖扩展语义的 SFTP Rename；同名移动目标会返回明确冲突，同连接移动同时使用 no-clobber 命令，跨连接复制到新目标时使用排他创建，避免静默覆盖文件或合并目录。目录列表、上传、在线读写和传输会复用共享 SSH 客户端，减少重复握手成本；在线文本读写创建 SFTP 客户端失败时会丢弃旧共享连接并重试一次，避免旧连接失效后拖慢或直接失败；`OpenTextFileStream` 在解析、stat 和打开后返回带元数据的可关闭原始读取器，HTTP 请求取消时可并发关闭远程文件与对应 SFTP 会话以打断未完成读取。旧 NDJSON 文本读取仍按 32 KiB 回调 chunk 供兼容接口使用，当前编辑器改用原始字节响应。复制粘贴会为复制操作的同目录或同名冲突生成唯一目标名，例如 `name copy.ext`，并阻止目录复制/移动到自身或子目录；删除会拒绝空路径、根路径和不明确的相对路径。目录列表会用 SFTP `RealPath` 解析请求路径，返回解析后的路径，包含 mode 和 UID:GID owner 元数据，并按文件夹优先排序。

## 已知工作

后端路径 API 需要继续同时支持 `~` 和 `/`。深层目录、大文件传输、权限拒绝路径、文本保存权限、远端 `cp`/`mv`/`rm` 可用性和是否解析友好用户名，都需要真实服务器验证。
